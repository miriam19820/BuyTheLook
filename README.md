                                                                                                                                                                                BuyTheLook - AI Personal Styling Engine
מערכת המלצות לאופנה שמשלבת לוגיקה פשוטה ומדויקת יחד עם AI שנותן את הטאץ' האישי. המטרה היא להציע פריטים שבאמת מתאימים למשתמשת לפי התקציב והטעם שלה.
1. גישה כללית לפתרון
בניתי מנוע שעובד בשני שלבים: קודם כל לוגיקה שלי (דטרמיניסטית) שבוחרת את הפריטים הכי רלוונטיים לפי הנתונים, ואז ה-AI נכנס לתמונה רק כדי לכתוב הסבר יפה ואישי לכל פריט. זה מאפשר למערכת להיות מהירה מאוד כי היא לא צריכה לשלוח את כל המאגר ל-AI בכל פעם.
2. הוראות הרצה
הורידי את הפרויקט למחשב.
בתיקיית ה-server, צרי קובץ .env והוסיפי את המפתח שלך: GEMINI_API_KEY=your_api_key.
הרצת השרת: בתוך תיקיית ה-server הריצי npm install ואז npm run dev.
הרצת האתר: בתוך תיקיית ה-client הריצי npm install ואז npm run dev.

3. החלטות עיצוב מרכזיות
שיטת הניקוד (Scoring): במקום "לזרוק" הכל על ה-AI, נתתי לכל פריט ניקוד לפי התאמה לאירוע, לסגנון ולצבעים. ככה התוצאות הכי טובות תמיד עולות למעלה.
טיפול בשגיאות: השתמשתי ב-Zod כדי לוודא שמה שהמשתמשת מכניסה הוא תקין. אם חסר מידע, היא מקבלת הודעה ברורה במקום שהאתר ייפול.
הסטאק: בחרתי ב-Node.js, TypeScript ו-React כי הם הכלים שאני מכירה הכי טוב ומאפשרים לי לכתוב קוד מסודר שלא עושה תקלות.
4. שימוש ב-AI
השתמשתי ב-Gemini  כדי לכתוב הסברים אישיים למשתמשת.
החלטה שלקחתי: במקום לתת ל-AI להחליט אילו פריטים להמליץ (מה שהיה הופך את המערכת לאיטית ויקרה), השתמשתי בו רק כדי "לתרגם" את ההמלצה למשפט נחמד. ככה הדירוג נשאר בשליטה שלי והוא גם מהיר מאוד.
5. שאלת סקייל
אם היו פה 100,000 פריטים ומיליוני משתמשות:
מסד נתונים: אי אפשר היה להחזיק את הכל בזיכרון, אז הייתי עוברת ל-Vector DB (כמו Pinecone) בשביל חיפוש מהיר.
ביצועים: הייתי משתמשת ב-Redis בשביל לשמור תוצאות של המלצות נפוצות, כדי לא לחשב אותן מהתחלה בכל פעם.
תורים (Queues): הייתי מריצה את חישוב ההמלצות ברקע כדי שהמשתמשת לא תחכה אם יש עומס על השרת.
